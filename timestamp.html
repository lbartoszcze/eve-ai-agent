📚 About Unix Timestamps

What is a Unix timestamp? A Unix timestamp (also called Epoch time, POSIX time, or Unix Epoch) counts the number of seconds elapsed since January 1, 1970, 00:00:00 UTC. This is a standard way computers track time, used in databases, APIs, log files, and virtually every programming language.

Seconds vs. Milliseconds: Traditional Unix timestamps use seconds (10 digits, e.g., 1700000000). JavaScript and some APIs use milliseconds (13 digits, e.g., 1700000000000). This tool auto-detects or lets you toggle manually.

Year 2038 Problem (Y2K38): 32-bit systems store Unix timestamps as a signed 32-bit integer, with a maximum value of 2147483647 (January 19, 2038 at 03:14:07 UTC). After this, 32-bit timestamp counters will overflow. Modern 64-bit systems are not affected.

ISO 8601: The international standard date format YYYY-MM-DDTHH:mm:ss.sssZ is unambiguous, sortable, and timezone-aware. It is the preferred format for APIs, JSON, and data interchange.
