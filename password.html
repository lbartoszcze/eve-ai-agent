Password Security Tips

Use 16+ Characters
Longer passwords are exponentially harder to crack. Each additional character multiplies the number of possible combinations. Aim for at least 16 characters for important accounts.
Mix Character Types
Combine uppercase letters, lowercase letters, numbers, and symbols. Using all four types dramatically increases the character pool and makes brute-force attacks much slower.
Never Reuse Passwords
Each account should have a unique password. If one service is breached, reused passwords let attackers access all your other accounts. Use a password manager to handle this.
Use a Password Manager
Password managers like Bitwarden, 1Password, or KeePass store all your passwords securely. You only need to remember one master password. They can also auto-fill login forms.
Enable Two-Factor Authentication
Even a strong password can be compromised via phishing. 2FA adds a second layer (like a phone code or authenticator app) so a stolen password alone is not enough.
Avoid Personal Information
Never use your name, birthday, pet's name, or common words. Attackers use dictionary attacks and social engineering to guess passwords based on personal info.
Check for Breaches
Use services like Have I Been Pwned to check if your email or password has appeared in a data breach. If it has, change the password immediately.
Rotate Sensitive Passwords
For critical accounts (banking, email, admin panels), change passwords periodically. Immediately change any password if you suspect it has been compromised.

How Password Strength Is Measured

Password strength is measured in bits of entropy. Entropy represents the number of guesses an attacker would need, on average, to crack a password through brute force. The formula is simple: entropy = length x log2(pool size), where pool size is the number of possible characters.

Understanding Character Pool Size

  • Lowercase only (a-z): 26 characters
  • Add uppercase (A-Z): 52 characters
  • Add numbers (0-9): 62 characters
  • Add symbols (!@#$...): 90+ characters

A 12-character password using all character types (pool of ~95) has about 79 bits of entropy. The same length using only lowercase letters has just 56 bits. That difference means the all-character password is over 8 million times harder to crack.

How Long Does It Take to Crack a Password?

The time to crack a password depends on the attack speed. Modern GPUs can test billions of hashes per second. At 10 billion guesses per second:

  • 8-character lowercase only: cracked in about 2 seconds
  • 8-character mixed (upper, lower, numbers, symbols): about 19 hours
  • 12-character mixed: about 2,000 years
  • 16-character mixed: about 500 million years
  • 20-character mixed: about 100 trillion years

Why Random Is Better Than Clever

Humans are bad at randomness. Patterns like "P@ssw0rd" or "Summer2024!" feel complex but are among the first things attackers try. Dictionary attacks, common substitution rules (a to @, e to 3), and leaked password databases make human-chosen passwords far weaker than their length suggests.

A truly random password generated by a cryptographically secure random number generator (like this tool uses via crypto.getRandomValues()) has no patterns, no dictionary words, and no predictable structure. This forces attackers into pure brute force, which is the hardest possible attack.
